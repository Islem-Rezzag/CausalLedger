# Release Ladder

This ladder describes intended public version meaning. It is not an implementation claim.

## v0.1.0: M00 Repo Operating System Foundation

The repo control plane exists and is tagged as the foundation release. No product functionality is implemented.

## v0.2.0: M01-M02 Domain And Local Development Foundation

Domain language, scope, local development structure, and baseline tooling are ready for product implementation.

## v0.3.0: M03-M06 Financial Truth Core

Canonical events, ledger primitives, provider/bank simulator, and deterministic invariants establish the first financial truth core.

## v0.4.0: M07-M09 Incident Digital Twin Core

Incident lifecycle, causal graph, and replay/digital twin behavior form the first incident reconstruction core.

## v0.5.0: M10-M13 Safe Agentic Layer

Agent tool contracts, advisory investigation, repair proposals, and human review preserve the safety boundary while enabling agentic workflows.

## v0.6.0: M14-M15 Benchmark And Demo

MoneyFlowBench and the UI command center support a credible benchmark-backed demo.

## v1.0.0: First Serious Public Product Release

The first public product release is useful, explainable, demoable, and honest about scope. It does not require all M16-M21 work.

## v1.1.0+: Connectors, Observability, Security, And Polish

Later minor releases expand connector coverage, observability, hardening, production polish, and public launch assets.

## v2.0.0: Company-Grade / Enterprise Version

The company-grade version adds enterprise controls, compliance posture, operational maturity, and commercial packaging.
