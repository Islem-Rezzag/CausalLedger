# M02 Monorepo and local development

## Milestone goal

Create the runnable local development foundation.

## Why it matters

A predictable local environment is required before product packages can be safely built.

## Deferred work

- `apps/agent-runtime` creation moved to the M10 era. Rationale: agent runtime should not exist before deterministic evidence, MoneyEvent, invariant, incident, graph, replay, and repair boundaries exist.
- Redis is deferred until needed. Rationale: no queue or scheduler should be added before the worker or orchestration design proves the need.

## Acceptance criteria

- Local checks run consistently.
- Safety boundaries remain consistent with `AGENTS.md` and active docs.
- Scope is limited to this milestone and its active plan.
- Validation evidence is recorded in the active plan and status docs.
- Handoff notes identify what changed, what did not change, and the next safe thread.
