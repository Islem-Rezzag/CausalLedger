# Milestone Closeout Prompt

Read active docs and the completed milestone plan. Verify completed submilestones, merged PRs, validation evidence, changed docs, changed code, remaining gaps, risks, and next milestone readiness. Use `docs/ops/github-pr-and-issue-workflow.md` for PR merge and post-merge tracking expectations.

Do not start the next milestone. Produce a closeout packet and move plans only when closeout criteria are satisfied.
